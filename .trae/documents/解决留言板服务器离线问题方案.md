# 关于“服务器目前离线”的问题说明与解决方案

## 现状分析
主公，此方世界中的“留言板”、“赛道排行榜”以及“全局饼干计数”等功能，皆需一个后端服务器来实时同步数据。
经过微臣查证，原作者 Bruno Simon 在源码中明确提到：**出于安全与隐私考虑，并未公开后端服务器的代码**。因此，目前前端因找不到对应的 WebSocket 信号，故显示为“离线”。

## 应对方案

### 方案一：隐藏离线警告（推荐，若无需多人互动）
若主公仅打算将此作作为个人展示，无需他人在留言板留下“悄悄话”，微臣建议直接隐藏该警告信息，以保界面整洁。
- **操作**：在 CSS 或 HTML 中将对应的 `offline-element` 设为不显示。

### 方案二：自行搭建后端（高阶玩家）
若主公执意要开启留言板功能，则需自行编写一个 WebSocket 服务器。
- **技术要点**：
    - 通信协议：使用 WebSocket。
    - 数据编码：需配合 `msgpack-lite` 进行二进制编码。
    - 接口要求：服务器需能处理 `whispersInsert`（插入留言）、`init`（初始化数据）等特定消息。
- **配置**：在 `.env` 文件中设置 `VITE_SERVER_URL` 指向您的服务器地址。

### 方案三：伪装在线状态（仅作展示）
若主公只想在截图或录屏时让界面显示为“在线”，微臣可修改代码逻辑，强行将连接状态设为 `true`。

## 建议
微臣建议先行采取**方案一**，待主公日后寻得良匠（或由微臣代劳）构建出后端秘法后，再行开启。

主公意下如何？若需微臣隐藏那碍眼的“离线”提示，请尽管吩咐！
